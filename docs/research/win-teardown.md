### 9. Secure input / elevation (OBSERVED message; INFERRED semantics)
- `SecureInputStatus` / `SecureInputStatusPayload` is a real shared IPC message (present in the helper schema and the main-bundle quicktype type table). On Windows this maps to detecting **elevated (run-as-admin) foreground windows** where a non-elevated LL keyboard hook + SendInput cannot inject (UIPI/integrity-level barrier) — INFERRED, the Windows analog of the Mac "Secure input is blocking keyboard shortcuts" (`EnableSecureEventInput`) handling. ### Implications for WhimprFlow (dual-platform clone)
- Wispr's Windows and Mac builds are **the same Electron 42.3.0 + same webpack JS bundle (identical commit hash)**, diverging only in: (a) the Electron native shell, (b) a per-platform **native helper** (C#/.NET NativeAOT `windows-helper-app` vs Swift `swift-helper-app`) that implements the identical stdio-JSON IPC schema, and (c) a handful of platform native node addons (`win-ca`/crypt32 vs `mac-ca`; Jabra per-OS). This validates the WhimprFlow "one codebase, native helper per OS" plan — though note Wispr uses **Electron**, not Tauri, and hand-writes two separate native helpers rather than sharing Rust (contrast Handy's single Tauri+Rust). If you keep Electron, mirror this: Squirrel.Windows (per-user, `%LocalAppData%`) + Squirrel.Mac; Electron `autoUpdater` with a `${channel}/${platform}/${arch}` feed; a native helper doing WH_KEYBOARD_LL hook + SendInput/clipboard paste + UIA on Windows, CGEvent tap + NSPasteboard + AX on Mac; Electron `Tray` for the tray; Azure Trusted Signing is a cheap path to a signed Windows binary.
- Windows text-injection recipe to replicate: set clipboard (with an HTML format option via `RegisterClipboardFormat`/`RenderHtmlText`), save+restore prior clipboard, then `SendInput` Ctrl+V (fallback `ShiftInsert`), with `FailedPasteNotification` detection and a "don't paste while modifier keys still down" guard — all in the helper, exactly as on Mac.
- Windows global PTT recipe: LL keyboard hook (not RegisterHotKey) + a `WindowsKeyUpSimulation` to suppress the Start menu when the Win key is the trigger; handle elevated-window/UIPI degradation (`SecureInputStatus`).
